# Auth0 Authentication Requirements

## 1. Overview & Architecture

### 1.1 Authentication Strategy

The Knight Platform uses a hybrid authentication architecture:

- **Auth0**: Handles password storage, authentication, and MFA enrollment/verification
- **Local OTP Service**: Handles email-based OTP for First-Time Registration (FTR) and password reset flows
- **Email Delivery**: Currently AhaSend, future migration to Salesforce API

### 1.3 Authentication Methods

| Method | Purpose | Storage |
|--------|---------|---------|
| Password | Primary authentication | Auth0 |
| Guardian Push | MFA (mobile push notifications) | Auth0 |
| TOTP | MFA (authenticator apps) | Auth0 |
| OTP | Email verification (FTR, password reset) | Local (Redis) |

**Note**: Passkey authentication is NOT supported. All passkey-related code should be removed.

### 2.3 Key Principles

1. **login_id and email are always email format** - They contain the same value
2. **Auth0 stores login_id as email** - This is the identifier used for Auth0 login
3. **OTP emails go to email field** - Not through Auth0's email system
4. **Auth0 handles password + MFA only** - No email delivery from Auth0

## 3. First-Time Registration (FTR) Flow

### 3.1 Prerequisites

- User record created by admin (status: `PENDING_ACTIVATION`)
- User receives registration email with unique link
- Link contains token for identifying the user

### 3.3 FTR Abandonment Handling

If a user abandons FTR mid-flow and returns later:

| Scenario | Action |
|----------|--------|
| User never verified email | Start from Step 2 |
| User verified email but no password | Reset `email_verified = false`, restart from Step 2 |
| User set password but no MFA | Skip to Step 5 (MFA enrollment) |

**Key principle**: If `password_set = true`, the password is already in Auth0. Don't recreate it - skip to MFA enrollment.

### 3.4 OTP Configuration

| Parameter | Value |
|-----------|-------|
| Code length | 6 digits |
| Expiration | 120 seconds |
| Max attempts | 3 |
| Rate limit | 3 requests per 60 seconds |
| Purpose codes | `ftr_email_verify`, `password_reset` |

```
POST /api/v1/mfa/enroll-guardian

3. User scans QR code with Auth0 Guardian app
4. Send test push notification
5. User approves test push
6. Set mfa_enrolled = true, mfa_preference = GUARDIAN
```

### 6.3 TOTP Enrollment

```
POST /api/v1/mfa/enroll-totp

1. Get TOTP secret from Auth0
2. Return QR code data for authenticator app
3. User scans with Google Authenticator/Authy
4. User enters verification code
5. Verify code against Auth0
6. Set mfa_enrolled = true, mfa_preference = TOTP
```

## 8. Admin Functions

### 8.1 MFA Reset Endpoints

**Indirect Client Admin**:
```
PUT /api/v1/indirect/users/{userId}/reset-mfa

Authorization: Requires ADMIN role for the indirect client
Effect: Sets allowMfaReenrollment = true for the user
```

**Direct Client Admin**:
```
PUT /api/v1/client/users/{userId}/reset-mfa

Authorization: Requires ADMIN role for the profile
Effect: Sets allowMfaReenrollment = true for the user
```

### 8.2 Authorization Rules

| Actor | Can Reset MFA For |
|-------|-------------------|
| Indirect Client Admin | Users in their indirect client only |
| Direct Client Admin | Users in their profile only |
| Bank Employee (Employee Portal) | NOT permitted |

## 9. Email/OTP Service

### 9.1 Current Implementation: AhaSend

The OTP service uses AhaSend for email delivery.

### 9.2 OTP Configuration

| Parameter | Value |
|-----------|-------|
| Code format | 6 numeric digits |
| Code expiration | 120 seconds |
| Max verification attempts | 3 |
| Rate limit | 3 OTP requests per 60 seconds |
| Storage | Redis with TTL |

### 9.3 OTP Purposes

| Purpose Code | Use Case |
|--------------|----------|
| `ftr_email_verify` | FTR email verification |
| `password_reset` | Forgot password flow |

### 9.4 Email Templates

**Email Verification (FTR)**:
```
Subject: Verify Your Email - Knight Platform

Your verification code is: {CODE}

This code expires in 2 minutes.

If you did not request this code, please ignore this email.
```

**Password Reset**:
```
Subject: Password Reset - Knight Platform

Your password reset code is: {CODE}

This code expires in 2 minutes.

If you did not request a password reset, please contact support.
```
